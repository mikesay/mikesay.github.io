【原】Git随笔(2) 从Git仓库中永久清理脏数据

发表于 2020-09-26 | 更新于 2020-12-14
本文字数： 3.9k | 阅读时长 ≈ 4 分钟

在开发代码的过程中，有时会不小心将不该提交的文件提交到Git仓库中，比如编译产生的临时二进制文件（忘了添加.gitignore），或者包含账号，密码等敏感信息的文件。临时的二进制文件放在Git仓库中没有意义，而且如果频繁改动的话，也会导致Git仓库逐渐变大，敏感信息会导致信息泄露且不符合信息安全标准，这些不该提交的文件或内容被称为Git仓库的脏数据，需要被清理掉。重新提交一个新的变更来清理这些脏数据是远远不够的，因为从历史版本中仍然能够找到它们。本文将介绍如何使用开源工具BFG Repo-Cleaner从Git仓库的变更历史中永久清除这些脏数据。

如果不借助第三方工具，要实现从Git仓库中永久清理脏数据，就需要使用Git的高级命令“git-filter-branch”重写历史记录。”git-filter-branch”功能强大，但是也相对复杂，所以就出现了第三方的开源工具封装”git-filter-branch”并提供简单易用的接口，BFG Repo-Cleaner就是其中比较出色的一个。仅管BFG Repo-Cleaner没有”git-filter-branch”的功能强大，但是它相对简洁、高效，而且基本上能满足日常大部分的需求。

使用BFG Repo-Cleaner

BFG Repo-Cleaner主要有以下两种类型的应用：

  • 替换文件中的敏感信息
  • 删除文件或文件夹

BFG Repo-Cleaner缺省不会清理最新版本代码里的脏数据，因为你的最新代码可能已经部署到生产环境，而且当前的部署或运行会依赖于这些已经存在的敏感信息或文件，所以，你需要手动地清理这些敏感信息或文件并修复部署和运行中可能存在的错误。如果你确认清理最新版本代码里的脏数据对部署和运行不会产生影响，你可以在命令中显式地加上参数“–no-blob-protection”。
